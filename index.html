01

这套系统是什么

不是从零设计的规范，是从两份真实制品里考古出来的语法

「知到 ZhiDao」设计系统把 PRD 文档与分享 deck 两份成品拆到像素与 XML 级，提炼成可迁移、可复用的一套语法：色板、字号、版式原型、母题、文案声线，每个数值都能回溯到源文件的具体页码。

定位

它服务的场景是「一人 + AI」——你说一句「用知到风做 XX」，AI 照着这套规范产出制品，不再从旧文件里现扒规格，也不再自由发挥。

v1.1

首版发布当天经四人格对抗性评审（视觉 / UX / PM / 挑刺用户），44 条发现零驳回，29 条去重后全部修复：对比度错值更正、深面 token 补覆写、deck 工具函数落地、发行件检验另有一轮。当档案读可信，当作业指导书用也不再翻车。

2.1两个统一锚点

跨载体身份的技术根基，换项目也别动：

  • 中文字体同源——deck 的「思源黑体」= web 栈里的「Noto Sans SC」，同一款 Source Han Sans 的双发行名。
  • 字号数字孪生——11.5–31 区间内 web 的 px 与 deck 的 pt 同数值同职责：正文 16px ↔ 15–16.5pt，页题 clamp 24–31px ↔ 29pt，元信息 11–12.5px ↔ 11.5–12.5pt。
03

六条设计原则

从制品中反推出来的，不是先验教条

P1

编辑部的秩序，科技的氛围

骨架永远是编号、字典表、左对齐、细分隔线；氛围交给深底、辉光与插画。秩序件从不发光，氛围件从不承载信息。

P2

骨架恒定，参数自由

页眉三件套、版心、编号体系从不漂移；渐变停靠点、插画逐页自由。先锁骨架，再谈自由。

P3

金色即结论

绿是过程与骨架，金只盖章：金句框金边、结论句金字。金一多，结论就没了。

P4

描边：颜色为主通道

观众分不出 0.1pt 差——区分语义的是颜色（中性/品牌绿/金），粗细只是制作规格三档。

P5

密度反比律

字越多图越退（表格页四角光斑），字越少图越进（金句页全幅）。信息密度和插画面积此消彼长。

P6

mono 标签语法

所有系统信息走 JetBrains Mono + 大写 + 宽字距，字号越小字距越大。这层机器感是 AI Native 的身份线索。

4.1三条任务路由

任务一 · HTML 文档

纸感编辑部风

读总纲 + layout-components

  • 引入 tokens.css + zd-components.css，只用语义 token
  • 骨架五段式：顶栏 → hero → metastrip → 双栏 shell → footer
  • 深色嵌入块挂 data-mode="ink"
  • 静态自查五条 + 人工视觉核对
任务二 · Deck

深色科技风

读 layout-components + motifs

  • from tokens_deck import *，开工先跑字体检查
  • 三层三明治：插画 → add_veil() → 原生文字
  • 版式只从 15 原型（L1–L15）里选
  • COM 渲染逐页自检，压暗有硬判据
任务三 · 换品牌

迁语法不迁颜色

读总纲第七节 + color

  • 只改两个 token 文件顶部原始色板层
  • 语法与用量比是身份，别动
  • 换色后必跑 check_contrast.py --audit
  • 冲突规则处置要写进迁移说明
入口

05

硬规则速记

违反即不是知到风；具体比值以 color 分册为权威

  • 金色即结论——每页一个金色焦点；亮金只留收尾强调页
  • 描边语义靠颜色——中性 / 品牌绿 / 金各司其职；粗细只是制作规格三档（1.0 / 1.2 / 1.4pt）
  • 灰阶纪律——deck 文字灰只有两档，别自创中间档；✗ 负向分治：浅面陶土暖红，深面只压灰不发彩
  • 跨面禁忌——浅面松绿别上深底、深面灰别上纸底、琥珀别当浅底文字；#00B050 禁用（色板漂移）
  • 版式与页眉收口——只从 15 原型选；完整三件套是内容页体例，章封面用变体，破格页别硬挂面包屑
  • 密度反比律——字越多图越退，字越少图越进
  • 秩序与氛围不混——编号/表格/分隔线从不发光，插画/辉光从不承载信息
06

deck 工具链

v1.1 把「规格散文」变成了可调用的函数——别再手糊

压暗渐变纱是 deck 可读性的命门，而 python-pptx 公开 API 做不出带透明度的三停渐变。tokens_deck.py 已封装好 XML 注入，一个新 deck 的骨架代码长这样：

pythonnew_deck.py
from tokens_deck import *

assert_fonts_installed()   # 缺思源黑体显式报错，绝不静默落雅黑

# 每页三层三明治：底图 → 压暗纱 → 原生文字
add_veil(slide)            # 含 alpha 三停渐变，停靠点随画面亮度调
add_header(slide, "页题", "第三章", "工作流")
add_conclusion_bar(slide, "结论：先锁骨架，再谈自由")
  • 插画降级链——①有生图工具按模板生成（顺手产出 alt 文案）②用户供图先校验 ③都没有就 BG_DEEP 纯底 + 压纱 + 四角光斑，同样合规
  • 压暗硬判据——文字框投影区平均亮度 L>60/255 即判失败，回去调纱；不再自由心证
  • QA 收口——PowerPoint COM 渲染逐页自检：金焦点是否唯一、页眉三件套是否恒定、文字是否压在暗区
