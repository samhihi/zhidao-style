## 一、品牌 DNA：一个语法，三个色彩面

两份制品表面气质迥异（暖纸编辑部 vs 深夜科技），底层却是同一套语法。**连接点藏在 HTML 文档内部**：hero、代码窗、SVG 示意图、footer 四类深色嵌入块，就是深色模式在纸感载体上的预演。

| | ① 纸感浅色面 | ② 暖深面（文档嵌入块） | ③ 冷深面（演讲） |
|---|---|---|---|
| 底 | 暖纸 `#F5F3EC` | 暖黑 `#181A1F` | 冷蓝黑 `#03070E`–`#243441` 六阶 |
| 品牌绿 | 松绿 `#1E5A44` | 薄荷 `#6FBF95` | 青绿 `#16C79A` |
| 暖强调 | 琥珀 `#C8932E`（文字用深档 `#7D5A0D`；v1.0 曾用 `#946A10`，paper 面不达 AA 已弃，考古注释见 color.md） | 沙金 `#C9A86A` | 暖金 `#E0B84A`（亮金 `#FFC000` 只在临门一脚） |
| 负向 | 陶土 `#B5532F` | 玫瑰 `#E2A4A4`（仅代码） | 无实色负向：压灰 + ✗（唯一红=终端点 `#FF5F56`） |
| 文字 | `#181A1F`/`#565851`/`#6E706A` 三档（弱化档 v1.0 为 `#8A8C82`，对比不达标已加深） | 米白系四档 | `#E6EDF3`/`#8A97A3` **仅两档** |

**色彩语法（三面通用）**：中性压场 → 绿立骨 → 金点睛 → 红陶/压灰表负向。绿承担骨架与正向判断，金是稀缺资源（一页尽量只给一个金色焦点；原片主导倾向，S14/S26 有反例），负向永远不抢戏。

**两个统一锚点**（跨载体身份的技术根基）：

1. **中文字体同源**：deck 的「思源黑体」= web 栈里的「Noto Sans SC」（Source Han Sans 的双发行名）。
2. **字号数字孪生**：11.5–31 区间内 web 的 px 与 deck 的 pt 同数值同职责（正文 16px↔15–16.5pt、页题 clamp 24–31px↔29pt、元信息 11–12.5px↔11.5–12.5pt）；两端各自延伸（web 顶端 clamp 到 76/80px，deck 顶端 34/40.5/44pt）。

## 二、六条设计原则（从制品中反推）

1. **编辑部的秩序，科技的氛围。** 骨架永远是编辑部式的：编号、字典表、左对齐、细分隔线；氛围交给深底、辉光与插画。两者不混：秩序件从不发光，氛围件从不承载信息。
2. **骨架恒定，参数自由。** 页眉绿竖条 39/41 页（缺 S1/S41）、29pt 页题 33/41；完整三件套（竖条+页题+面包屑）约 32 页，是**内容页体例**；章封面用变体（加高竖条+章名+「第 N 章 · 共六章」进度语），S1/S2/S41 破格。版心/编号体系从不漂移；而压暗渐变的停靠点逐页手调、插画每页不同。先锁骨架，再谈自由。
3. **金色即结论。** 绿是过程与骨架，金只盖章：金句框 1.4pt 金边、结论句金字。金一多，结论就没了。
4. **描边：颜色为主通道，粗细为制作规格。** 投影距离上观众分不出 0.1–0.2pt，真正区分语义的是**颜色**（中性 / 品牌绿 / 金）；粗细是统一的制作规格，不是观众可感知的层级。三档定版：中性 1.0pt、品牌绿一律 1.2pt（含章封面高亮框）、金句 1.4pt。原片同一语义档内实测 1.2pt×3 页、1.3pt×6 页、S38 一处 1.0pt——三种粗细对应同一语义，恰证明起作用的通道是颜色；v1.1 收口取 1.2（历史测量保留为事实，见 color.md §5.5）。
5. **密度反比律。** 字越多图越退（表格页插画只在四角透光），字越少图越进（金句页图占 100%）。信息密度和插画面积此消彼长。
6. **mono 标签语法。** 所有「系统信息」（编号/眉题/徽章/表头/元数据）走 JetBrains Mono + 大写 + 宽字距，字号越小字距越大（.04em→.22em）；deck 侧无字距属性，宽字距靠手工空格（「第 一 章 · 共六章」）。这层「机器感」正是 AI Native 产品的身份线索。

## 三、速查 · 版式

### Web 文档骨架（自上而下）

```
sticky 顶栏(毛玻璃 rgba(245,243,236,.86)+blur10 + 2px 滚动进度条)
→ hero(暖黑底 + 双径向氛围光: 品牌绿 .32 右上 + 琥珀 .10 左下)
→ metastrip(四格元信息带, mono 大写小标签 + 粗体值)
→ shell 双栏(版心 1160px = 侧目录 236px + gap 54px + 正文)
   侧目录: sticky top78, 前导零编号 01–13, IntersectionObserver 高亮
→ footer(暖黑底, 与 hero 首尾呼应)
```

组件家族记三条规律即可（全表见分册）：**容器**（card/mini/story/method…统一 1px `--line` 边 + 10–14px 圆角 + `--surface` 底）、**标注**（call/changelog 左侧 3–4px 语义色强调条）、**徽章**（chip/pill/pchip/kicker，mono 小字 + 语义色变体 pine/amber/clay 三态同构）。

## 五、插画配方（deck 的独有资产）

风格要素：深夜蓝黑底 + 青绿主光/暖金辅光 ≈7:3 + 玻璃/拉丝金属/全息线框材质 + 写实 CG 渲染；**暗部 ≥60% 且文字必须压在暗区**；全幅图统一 2048×1152（4K 放映升 3840×2160）。成图后仍叠垂直压暗纱兜底——一律调用 `tokens_deck.add_veil()`（内部注入含 alpha 的三停渐变；双色 `#03070E`/`#06111A`，90° 为主，默认模板起步、停靠点按画面亮度逐页调——恒定的只有「双色」和「文字区必须压到可读」两件事），别手糊。无生图工具时的降级链见「迁移指南」。

**中文提示词模板**：

> 深夜蓝黑科技场景，主色调为深蓝黑（#03070E 至 #0C1418），青绿色（#16C79A）为主光源，暖金色（#E0B84A）为辅助光，比例约七比三。画面主体为【替换：全息玻璃数据平台 / 发光节点网络与悬浮文档卡 / 等距发光路径地图 / 机器人施工队沙盘】，材质为半透明玻璃层叠、拉丝金属与全息线框，表面有精细反射，写实 CG 渲染质感。【替换视角：等距视角 / 低角度全景微俯视 / 平视特写浅景深】。青绿辉光、光纤与霓虹描边，暖金粒子流与逆光轮廓，地面有反射光轨。画面【替换：左半 / 中部 / 右半】保持大面积暗部留白用于排版文字，暗部占比不低于百分之六十。电影感灯光，16:9，2048×1152。禁止出现文字、水印、大面积纯白。

英文版见 `reference/motifs.md` §3.3。全幅/半幅/点缀三档用法：低密度页全幅、论述页半幅（亮部让出 40–55%）、高密度页只留四角光斑。

## 六、速查 · 文案语法

- **句式配方**：对比句（「仓库，不是助理」）、箭头句（「答得出 → 用得出」）、大于号句（「编译 > 检索」）、口语断言（「跑就完事」）。共同声线：第一人称、有立场、反空话；文档端编辑部严谨，演讲端口语+现场感。
- **中英混排**：中文为主，英文只留给不可译术语/品牌名/文件名/公式；「·」间隔号做枚举分隔（deck 面包屑分隔=两个半角空格夹 U+00B7）。
- **信息架构骨架**：文档=编号章节(kicker)+小节(hx)+changelog 双卡+用户故事卡(含验收口径)+词汇表；演讲=目录页+六章封面(进度菜单)+章内「(N) 节名」面包屑+演示仪式页+章末带走卡+资料包兑现页（开场承诺 S4→S41 核销，首尾闭环）。
- **命名**：CSS 组件一律 `zd-` 前缀；语义态走 data 属性（`data-l="raw|notes|wiki|briefs"`）；文件名「日期-标题」。

## 七、迁移指南

### 新做一份「知到风」HTML 文档

1. 引入 `tokens.css` + `zd-components.css`，正文只引用语义 token（`--text-1`/`--brand`/`--bg-1`…）；组件几何别手写，`zd-components.css` 已从 PRD 源码抽出校验；交互状态（hover/焦点环/disabled）用 tokens.css 的最小状态集，别即兴
2. 搭骨架五段式：顶栏+进度条 → hero（暖黑+双径向光）→ metastrip → 双栏 shell → footer
3. 深色嵌入块（代码窗/示意图）切 `data-mode="ink"`，绿换薄荷、金换沙金
4. 所有「系统信息」上 mono 标签语法；列表符用菱形（6px 方块 rotate 45°）
5. 静态自查五条：琥珀文字必用深档 `#7D5A0D` / `--mute` 只做 11–12px 元信息 / 语义 token 之外无裸 hex / 深色块 `data-mode` 已挂载 / 列表符是菱形。视觉渲染核对是**人工步骤**：请用户打开核对

### 新做一份「知到风」deck

1. `from tokens_deck import *`；开跑先 `assert_fonts_installed()`——缺思源黑体显式报错，绝不静默落雅黑。画布 `SLIDE_W×SLIDE_H`（真 16:9）；只有改旧片才用 `SLIDE_W_LEGACY`（原片值系模板转换残留，勿仿）
2. 每页三层三明治：全幅插画（降级链见下）→ `add_veil(slide)` 压暗渐变纱（内部注入含 alpha 的三停 gradFill，默认后半场模板、停靠点随画面亮度调）→ 原生文字层。**别手糊渐变纱**——python-pptx 公开 API 做不出带 alpha 的三停
3. 内容页页眉三件套一律 `add_header(slide, title, chapter, section)`（单点维护，别逐页手放；chapter/section 缺省则只放竖条+页题）；章封面不挂面包屑，用变体（加高竖条+章名+进度语）；结论条（S38 体例：面板底+品牌绿 1.2pt 边，非金框——金框专属金句）用 `add_conclusion_bar(slide, text)`；文字一律 `set_font()` 三槽齐设
4. 版式从 15 原型（L1–L15）选，不发明新版式；表格/口诀用 L13 原生表格（改文案不求人），华丽信息图才用 L9 烘焙（接受文字不可改）
5. 金色纪律：每页一个金色焦点；`GOLD_HOT` 只留给收尾两页

**插画降级链**（三条路都合规，别卡死也别自由发挥）：

- ① 有生图工具 → 按 motifs 分册 §3.2/3.3 提示词模板生成，**同时产出一句 alt 文案**
- ② 用户供图 → 校验 2048×1152 与暗部占比，过了才用
- ③ 都没有 → `BG_DEEP` 纯底 + `add_veil()` + 四角光斑——这也是合规的知到风（表格/高密度页本就只需四角透光）

**压暗 QA 判据**（不再自由心证）：对每个文字框投影区取渲染图平均亮度，L>60/255 即判失败，回去调 `add_veil` 停靠点（PIL 可算）；S38 属违例，新项目禁止复制。

### 无障碍三条硬规则（两种载体都算数）

1. 每张插画/示意图必带 alt 或 aria-label（生图时顺手产出一句 alt 文案，见降级链①）
2. heading 层级不跳级
3. 平滑滚动必配 `prefers-reduced-motion` 兜底（样板在 tokens.css）

### 换项目时，什么可换、什么是身份

| 可换 | 是身份，别动 |
|---|---|
| 具体色相（绿可换蓝紫系） | 「中性压场→主色立骨→暖色点睛」的语法与用量比 |
| 插画主体（城市/地图/沙盘） | 暗部 ≥60%、文字压暗区、主辅光 7:3 |
| 章节数、版式组合 | 页眉三件套、骨架恒定参数自由、密度反比律 |
| 展示字体（Space Grotesk 可换） | mono 标签语法、中文黑体双载体同源 |

换色不是改个 hex 就完事，**两步硬流程**：

1. **换色后必跑** `py check_contrast.py --audit`。阈值表：正文 4.5:1 / 大字 3.0:1（仅 ≥24px regular 或 ≥18.66px bold 算大字）/ 深底强调 7.0:1。不过线就加深色档，别放行
2. **冲突规则处置声明**：主色若占了蓝紫系，「蓝=外部引用/技术中性」的修辞怎么重分配（外部引用色改用什么）、暖强调是否保金——写进目标项目的迁移说明。不写=没换完

另注：淡染 α 档与光晕 rgba 是「同色相公式」（按新主色重算），不是可照抄的字面量。

### 其他载体（第三种产物往哪走）

| 产物 | 裁定 |
|---|---|
| 独立深色载体（海报/长图/封面） | 冷深面（deck 色板） |
| 嵌入浅色环境的块（周报/briefs 里的卡片） | 纸感面 + 暖深嵌入块 |
| 纯文本产物（Telegram 消息/digest） | 不带视觉，只带 voice-ia 的声线与句式 |

跨载体通用母题取「十件套」前五件：左强调条、双轨编号、→ 箭头链、✓/✗ 双色判定、胶囊徽章。

## 八、已知设计债（沉淀规范时的定版决定）

原片中实测发现、新项目应避免的漂移（已修复的标「已解决」，历史测量一律保留为事实）：

1. **双金未统一**：`#E0B84A` 与 `#FFC000` 并存 → 定版：常规金 `E0B84A`，亮金 `FFC000` 仅收尾强调
2. **`#00B050` 漂移**：全片仅两处（S2 徽章边/S33 卡边），与品牌绿对比仅 1.32:1 → 定版：禁用，统一 `16C79A`
3. **L9 烘焙图 7 页文字不可改**（S15 已归 L14 变体）：视觉华丽但维护成本高 → 定版：可改文案优先原生排版/L13 表格
4. **弱化色对比度压线**：web `--mute` 原值 `#8A8C82` 3.07:1、`--clay` 4.46:1，deck `#8A97A3` 混排段落用 → **mute 已解决（v1.1）**：加深为 `#6E706A`（paper 面过 AA，实算归 color.md），用途仍限 11–12px 元信息、不承载正文；同时废除「≥18px 可豁免」的错误豁免线（WCAG 大字线是 ≥24px regular / ≥18.66px bold）。clay 与 deck 压灰维持「只做短元信息，不承载正文」
5. **章封面小注文案双版本**（「定义力，才稀缺」4 页 vs「想清楚，才稀缺」2 页）→ 定版：锁定**「定义力，才稀缺」**，同一 deck 内固定用这句
6. **烘焙图尺寸杂**（2048×1152 为主，另有 1672×941 等）→ 定版：全幅统一 2048×1152；4K 放映升 3840×2160
7. **渐变纱参数前半场逐页手调无模板** → **已解决（v1.1）**：tokens_deck.py 新增 `add_veil()`，以 lxml 注入含 alpha 的三停 `a:gradFill`（python-pptx 公开 API 做不出三停带 alpha），默认后半场模板（顶 100%/中 pos≈69% α≈25%/底 84%）为起点、随插画亮度微调
8. **母版化未做**：41 页页眉三件套全部逐页手放（rels 全指 blank layout），改一次页眉规格要动 39 处 → v1.1 以 `add_header()` 把逐页手放收敛为**单点维护**；真正的 slideLayout 母版化留给 v2
