## 0. 结论先行

1. **这是一个品牌、两种模式,不是两套设计。** 浅色纸感(HTML 主体)与深夜科技(PPT 全部)共享同一条色彩语法:**中性底压场 → 品牌绿立骨 → 金色点睛 → 红陶/压灰表负向**。两载体的强调公式、用量配比、语义分工逐条对得上。
2. **HTML 的深色嵌入块(hero/code/diagram/footer)就是两载体的连接点**:它在纸感文档内部预演了整套深色模式——深底 + 暖白正文 + 品牌绿亮档(#6FBF95)+ 金色亮档(#C9A86A)+ 弱灰注释,与 PPT 的 #03070E + #E6EDF3 + #16C79A + #E0B84A + #8A97A3 是同一公式的两次实例化。
3. **两个深色面不是同一个深色**:HTML 深区是「墨底暖灰」(绿倾暖灰阶,延续纸的温度),PPT 是「深夜蓝黑」(蓝倾冷灰阶)。沉淀 token 时要承认这是**两套深色中性灰阶、一套语义映射**。
4. **深色模式没有现成的一对一负向色**(clay 的对位缺失):HTML 深区只在代码语法里用玫瑰 #E2A4A4,PPT 原生层零负向实色(负向靠压灰 #8A97A3、位图内暗红 ⊗、❌ emoji)。本册提名 **#E2A4A4 为 neg 的深色档**(对比度 9.69:1/7.96:1,双底达标),作为系统补全项。
5. **对比度审计主要风险点两个(v1.1 已双双定版修复)**:浅色模式 mute 原值 `#8A8C82` 仅 3.07:1,v1.1 定版加深为 `#6E706A`(≈4.52:1,11–12px 元信息现已合规);amber 上纸底作字原以深档 `#946A10` 补偿(原档 `#C8932E` 仅 2.47:1),但 `#946A10` 对 paper 也只有 4.37:1——v1.1 定版文字档为 `#7D5A0D`(5.66:1 AA ✅),`#946A10` 降为考古注释。深色两面的正文、强调色全部 AA 达标,PPT 大字甚至大面积 AAA。

## 1. 架构:一个品牌、两种模式

### 1.1 双面结构

```
知到品牌色彩系统
├── 纸感浅色模式(paper mode)——HTML PRD 主体
│     底:米纸 #F5F3EC   字:深墨 #181A1F   强调:pine/clay/amber 原档(低明度、可上纸)
└── 深夜科技深色模式(night mode)
      ├── HTML 深色嵌入块(hero/code/diagram/footer)——墨底 #181A1F、暖灰阶
      └── PPT 全部 41 页——蓝黑底 #03070E~#243441、冷灰阶
```

### 1.2 为什么说深色嵌入块是连接点(论证)

| 证据 | HTML 深色嵌入块 | PPT 原生层 | 结论 |
|---|---|---|---|
| 深底 + 黑描边/压暗层 | `#181A1F` 底 + `#000` 描边(hero/code/diagram;footer 无描边) | `#03070E→#06111A` 渐变压暗层 | 同为「近黑深底上做戏」 |
| 主文字换暖白/近白 | `#EFEDE6`/`#D8D6CE` | `#E6EDF3` | 同为「不用纯白的高级白」 |
| 弱化字换低饱和灰 | `#9A9D95`/`#7E8479` | `#8A97A3` | 同为「灰阶即层级」 |
| 品牌绿升亮才上深底 | pine → 薄荷 `#6FBF95`(文件名/链接/图标) | pine → 青绿 `#16C79A`(竖条/面包屑/描边) | **同一动作:绿色提明度抗深底**,只是抬升幅度不同 |
| 金色点睛 | 沙金 `#C9A86A`(字符串)/amber 原档(eyebrow、回环) | 暖金 `#E0B84A`(金句/敲黑板)+亮金 `#FFC000` | 同为「金=稀缺高光」 |
| 品牌实色只给核心 | SVG 唯一实色节点 `dg-core` 填 pine | 表头反白填 `#16C79A`、高亮框绿描边 | 同为「实色=看这里」 |
| 氛围光效 | hero 径向绿光晕 + 琥珀微光 | 插画青绿主光 + 暖金辅光 | 同一「绿主金辅」光语言 |

HTML 作者在纸面文档里凡是要表达「机器视角/系统内部」(首屏宣言、代码、架构图、页脚)就切入深色块——PPT 整场演讲都处在这个「系统内部」语境里,所以整场即深色。**深色嵌入块 = 品牌深色模式在浅色载体上的四个窗口**,迁移新项目时,它就是两种模式共存的现成样板。

### 1.3 两个深色面的差异(必须承认的分叉)

| 维度 | HTML 深色嵌入块 | PPT |
|---|---|---|
| 深底基调 | 墨色 `#181A1F`(中性偏暖) | 蓝黑 `#03070E`/`#06111A`(明显蓝倾) |
| 中性灰阶温度 | 绿倾暖灰(`#7E8479`/`#9A9D95`/`#C7C9C0`) | 蓝倾冷灰(`#8A97A3`/`#243441`) |
| 品牌绿亮档 | `#6FBF95`(薄荷,柔) | `#16C79A`(青绿,艳,饱和度高一截) |
| 金档 | `#C9A86A`(沙金,灰调) | `#E0B84A`(暖金)+`#FFC000`(亮金) |
| 成因推断 | 要与米纸底同页共存,深区必须留纸的暖度 | 独立放映 + AI 插画高饱和光效,需要更亮更纯的强调 |
| 系统化建议 | 定为 `dark-embed`(暖深)子面 | 定为 `dark-stage`(冷深)子面 |

## 3. 职责边界 · 用量权重 · 语义规则

### 3.1 三级用量结构(主导/支撑/点睛)

| 层 | 占比感 | 浅色模式 | 深色模式(PPT 实测计数佐证) |
|---|---|---|---|
| **主导(~85–90%)** | 底 + 中性文字 | paper/surface + ink/graphite/mute | 深底渐变 + `#E6EDF3`/`#8A97A3`(两色合计 559 次,占全部 srgbClr 引用 1040 次的 ~54%,过半;加品牌青绿三色合计 736 次,~71%) |
| **支撑(~8–12%)** | 品牌绿骨架 | pine:链接/选区/进度条/nav active/核心组件/soulbadge | `#16C79A`(177 次):每页必有竖条 + 面包屑章名,但**成块面积极小**——绿是骨不是肉 |
| **点睛(~1–4%)** | 金 + 红陶 | amber(修订/心法/回环)、clay(警示/编号/P0) | `#E0B84A` 43 次/18 页(一页一个金色焦点是**主导倾向**,多数页成立;已知例外见 3.2 金色条与 Don't#6);`#FFC000` 全片仅 2 页可见 |

### 3.2 语义规则(两载体通用)

- **绿 = 品牌/正向/核心/导航/「你在这里」/已验证。** 浅面:pine 实底只给最高荣誉(soulbadge、核心节点、now 标签);深面:绿描边=高亮框语法(制作规格 v1.1 收口为一律 1.2pt,原片 1.2/1.3 混用见 §5.5),绿反白底=表头。绿永远负责「结构与肯定」,不负责情绪。
- **金 = 警示/亮点/稀缺/金句/回环。** 一页一处是「音量键」式的**主导倾向**:HTML 里 amber 管修订中、心法、往回织回环;PPT 里 E0B84A 管敲黑板词与金句框,FFC000 只在收尾两页做「临门一脚」。实测例外:S26 有 3 处彼此独立的金色焦点(金句行、段首引导、卡头,E0B84A run×5),另 S14 run×4、S18/S19/S33/S36/S40 各 run×3。金的稀缺性本身就是设计资产——沉淀为规范时保留「一页一处」作目标纪律,但引用原作时别声称全片实测无例外。
- **红陶 = 负向/风险/对比项/原料层/编号。** 只在浅面有完整戏份(clay 章号、P0、bad 卡、raw 层、病根 call);深面负向靠**去饱和压灰**(旧概念整行 `#8A97A3`、旧世界黑白化)——「不给负向发彩色」是 PPT 的独特策略,批评对象连颜色都不配拥有。✗/反例记号同理按**载体分治**:浅面=clay 暖红系,深面=压灰 `#8A97A3`(S12「❌ AI 增强」run 实测即灰)——不存在「两载体零例外」的统一 ✗ 色,谁再这么写就是没看原片。
- **蓝 = 外部/技术中性/特殊话题。** 两载体都把蓝留给「非我方语境」:HTML 深区代码键名 `#7FB7E8`;PPT 引用页(S6)、合规页(S31)、MVP 隐喻页(S13)整图转蓝;S19 旧态画框用非品牌蓝灰描边(渲染 ≈`#385D8A`)对照新态品牌绿框——「旧=非品牌蓝灰、新=品牌绿」是有意表态。蓝从不进入页面骨架。
- **双绿分工(仅浅面)**:pine=结论/核心,moss=过程/次级肯定(小节号、P1、笔记层)。PPT 是单绿制,迁移时如需过程档,可从 `#16C79A` 降饱和派生(待定义)。
- **实色 = 最高级。** 两面共同铁律:平时用「染底+描边」,实色填充只给唯一核心(soulbadge/dg-core/表头/P0-P1 徽章)。

### 3.3 状态色约定(从 HTML 编年史沉淀)

修订中 = amber → 已验证/已落地 = pine → 风险/病根 = clay。这条「琥珀转绿」的状态叙事同样适用于深色面(PPT 结论条绿框 = 已确认,金句框金框 = 理念待内化)。

## 5. 透明度 · 渐变 · 光效使用规律

### 5.1 rgba 淡染三件套(浅色模式的强调公式)

**`实色文字 + rgba(品牌色, .04–.1) 底 + rgba(品牌色, .2–.35) 描边`** —— HTML 全部彩色强调(chip/call/sig/concept/loop-path)都由此派生,精确档位:
- pine `rgba(30,90,68,α)`:底 `.04/.05/.06/.08/.1`,描边 `.2/.25/.3/.35`,hero 光晕 `.32`,hero pill 底 `.18`
- clay `rgba(181,83,47,α)`:底 `.04/.05`,描边 `.25/.3`
- amber `rgba(200,147,46,α)`:光晕 `.10`,描边 `.3`
- moss `rgba(62,122,92,.5)`:hero pill.pine 描边
- 白 `rgba(255,255,255,.03)`:深底默认 pill 底(深面版「淡染」)

### 5.2 径向渐变氛围(HTML hero,唯一的光效)

```css
radial-gradient(680px 420px at 88% -10%, rgba(30,90,68,.32), transparent 60%),  /* 右上松绿光晕 */
radial-gradient(520px 360px at 6% 120%, rgba(200,147,46,.10), transparent 62%)  /* 左下琥珀微光 */
```
规律:**绿主金辅、对角布光、透明度绿 3 倍于金**——与 PPT 插画「青绿主光 + 暖金辅光」是同一布光公式。

### 5.3 线性渐变(两面各一类)

- 浅面「荣誉渐变」——共性是**绿意从一缘渐隐入 surface**,标记「核心/当前」,但参数按组件各配,并非统一:
  - wiki 层:`linear-gradient(90deg, rgba(30,90,68,.05), var(--surface) 40%)`(左缘渐隐);
  - concept.good:`linear-gradient(180deg, rgba(30,90,68,.04), …)`(顶缘渐隐);
  - road.now:`linear-gradient(180deg, rgba(30,90,68,.06), …)`(顶缘渐隐)。
- 深面「压暗渐变」(PPT 每页有且仅有 1 个背景 `gradFill`,可读性的命门),实测分两种形态:
  - **后半场模态配置(S23/27/30/31/33/34/35 七页逐值一致)**:`ang=5400000`(自上而下),顶 `#03070E`@100% → 中 `#06111A`@pos≈69%/alpha≈25%(透出插画)→ 底回深 @84%。「上下深、中间透」,页眉与底部结论永远可读,插画细节留在中带。注意这是**模态(众数)配置而非全覆盖固定模板**——其余后半场页中停 α 5–59% 随插画亮度逐页调(与 motifs 分册 §1.7 同口径);正确用法是以此配置为起点,按画面亮度微调中停。
  - **前半场逐页自由调参**:顶部 `#03070E` alpha 实测跨 0–100%(≥94% 的约 27 页;反例如 S1=80%、S2=63%、S5≈0%、S16=42%、S25=69%,S12/S13 整页无 `#03070E` stop);中段 `#06111A` pos 实测 13–89%、alpha 0–84%;底部 stop alpha 实测 0–98%。方向也有例外:**S7/S10/S20 用 `ang=16200000`(自下而上)反向渐变**。共同不变的只有一条:**结论文字所在端保持高不透明压暗**。

### 5.4 玻璃拟态参数

| 场景 | 参数 |
|---|---|
| HTML 粘性顶栏 | `rgba(245,243,236,.86)`(paper 的 86%)+ `backdrop-filter: blur(10px)` |
| PPT 截图画框(S19) | `#142028` @70% alpha 填充 + 描边表态(旧=accent1 蓝灰 2.25pt thickThin,渲染 ≈`#385D8A`;新=`#16C79A` 1.2pt) |
| PPT 徽章(S2) | schemeClr tx1 @57%(半透明黑玻璃)+ 1pt 绿描边 |
| PPT 常规容器 | `#142028` 近实底 + `#243441` 1pt「隐形描边」(对比 1.30:1,只给边缘一丝定义) |

### 5.5 描边表态:颜色是语义主通道,粗细是制作规格(v1.1 收口)

**语义靠颜色分层**:`#243441` = 中性容器 → `#16C79A` = 看这里/当前项 → `#E0B84A` = 金句盖章——颜色是每次都成立的区分通道。粗细随色配套,但 0.1–0.2pt 的档差在投影距离不可感知,v1.1 起**不再宣称粗细差本身是观众可读的层级**:粗细是制作规格,管的是「同档一致、不手抖」,不管表意。
**v1.1 定版三档:中性 1.0pt / 品牌绿一律 1.2pt(含章封面高亮框)/ 金句 1.4pt。**
考古事实(原片实测,保留为历史测量,勿据此改规范):品牌绿描边 1.2pt×3(S18/19/28)、1.3pt×6(S3/9/16/22/27/34)、S38 一处 1.0pt(与中性同粗,例外勿仿);章封面高亮框原片 1.3pt;偏差色 `#00B050` 描边 1pt(S2/S33)。同一语义在原片里漂出三种粗细,恰是「粗细当不成主通道」的证据——v1.1 收口取 1.2。

## 6. 迁移到新项目的最小集

最小集 = **直接引入 [`../tokens.css`](../tokens.css)**(三个色彩面齐备::root 纸感浅色面 / `[data-mode="ink"]` 暖深面 / `[data-mode="deep"]` 冷深面),组件样式只引用语义层变量、不写裸 hex。变量名以 tokens.css 为准,本册不再自创命名(v1.0 曾在此列过一套 `--zd-*` 名,与 token 文件是两套 API,v1.1 删除)。核心语义变量对照:

```
/* 同一变量名,三面各自解析(浅面 / 暖深 ink / 冷深 deep) */
--brand           品牌绿    pine #1E5A44 / 薄荷 #6FBF95 / 青绿 #16C79A
--brand-2         次级绿    moss #3E7A5C / 雾绿 #BFE3CF / =--brand(深面无独立次级绿)
--warm            金(块/描边) amber #C8932E / 沙金 #C9A86A / 暖金 #E0B84A(亮金 #FFC000 挂 --warm-hot,仅限临门一脚)
--warm-text       金色文字档  #7D5A0D(v1.1 定版,paper 5.66:1)/ #C9A86A / #E0B84A
--negative        负向      clay #B5532F / 玫瑰 #E2A4A4 / 玫瑰 #E2A4A4(终端红点是组件色,另挂 --dot-red)
--bg-0 / --bg-1   场 / 一级容器  paper·surface / #181A1F·#22262E / #06111A·#142028
--text-1/2/3      主/次/弱文字  --text-3 浅面即 mute(v1.1 #6E706A)
--border-1        常规分隔
--border-strong   强分隔/结构线(永远中性)
--border-accent   品牌强调描边(冷深面 #16C79A)
```

三条公式:
1. **淡染三件套**(浅面强调):实色字 + 同色 rgba(.04–.1) 底 + 同色 rgba(.2–.35) 描边。
2. **压暗渐变**(深面全幅图可读性):自上而下线性渐变,顶 `#03070E` 高不透明 → 中段透出插画 → 底回深(模态配置,七页逐值一致:顶 100%/中 pos≈69%、alpha≈25%/底 84%,见 §5.3);铁律是**结论文字所在端保持高不透明**,中段按插画亮度调参。
3. **描边表态**:中性 1.0pt 隐形边 → 品牌绿 1.2pt「看这里」→ 金 1.4pt「盖章」(v1.1 定版;语义靠颜色,粗细是制作规格,见 §5.5);实色填充只给唯一核心。

再加一条纪律:**金色一页/一屏只出现一处**——这是从原作主导倾向提炼的目标纪律(原作多数页成立,S26 例外),新项目按纪律执行。

## 7. Do / Don't

### Do ✅

1. **深浅互换永远走映射表**:pine↔teal(/薄荷)、amber 深档↔gold、clay↔rose——同语义、异色值,逐色重配。
2. **金色当音量键用**:一页一处、只给最想让人记住的词;暖金管理念,亮金只在收尾「临门一脚」(全片仅 2 页可见)。
3. **给负向做「降饱和」而不是「上大红」**:PPT 用整行压灰 `#8A97A3`、黑白化旧世界;要红时用 clay/rose 的低饱和档。
4. **实色填充留给唯一核心**:soulbadge、dg-core、表头反白、P0/P1 徽章——其余一律「染底 + 描边」。
5. **深底上的字先过 4.5:1 再上屏**:四彩点睛(薄荷/沙金/冷蓝/玫瑰)全部 7:1+ 是这套系统耐看的底层原因;新增强调色照此标准配。
6. **全幅图上放字,先铺压暗渐变**:结论文字所在端保持高不透明压暗(模态配置为顶部 100%,见 §5.3),中段按图的亮度逐页调 pos/alpha。
7. **蓝色专管「外部/特殊」**:引用外部观点、合规声明、代码键名才用蓝,让「换色」本身成为修辞;「旧态」也可用非品牌蓝灰描边对照品牌绿「新态」(S19 手法)。
8. **状态叙事用「琥珀转绿」**:修订中 amber → 已验证 pine/绿框,贯穿 changelog、路线图、结论条。

### Don't ❌

1. **别把 `#C8932E` 直接当浅底文字**(2.47:1)——浅底作字必换文字深档 `#7D5A0D`(v1.1 定版;原片用的 `#946A10`/`#9A6F12` 已降为考古,见 §4.1);同理 `#16C79A`、`#E0B84A` 严禁上纸底(1.96/1.70:1)。
2. **别沿用 mute 原值 `#8A8C82`**(3.07:1,v1.0 原值)——v1.1 已定版加深为 `#6E706A`(≈4.52:1),用途不变:11–12px 元信息。也别再引用「≥18px 装饰性元信息可豁免」——WCAG 大字线是 ≥24px regular / ≥18.66px bold,元信息一律按正文 4.5:1 要求;更重的弱化字用 graphite `#565851`。
