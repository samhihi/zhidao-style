# 知到 ZhiDao · 图形母题与插画系统（motifs.md）

> **本册角色**：《知到》设计系统 · 图形母题与插画分册（维度：图形母题 / 图标 / 插画 / 示意图 / 氛围）。
> **事实来源**：`docs/prd/zhidao-prd-v1.2.html`（载体 A）与 `docs/share/__back/《从0到1，用AI Agent搭一个能跑的产品》.pptx`（41 页，载体 B）。
> **校验方式**：逐断言溯源核查——HTML 逐行比对源码、PPT 41 页 slide XML 全量扫描 + 渲染图抽查 + EMU/pt 换算重算（2026-07-14）。
> **上级文档**：`../design-system.md`
> **载体代号**：**A = 浅色纸感 HTML PRD**（暖纸底编辑部风），**B = 深色科技风 PPT**（41 页演讲 deck）。

---

## 0. 结论先行

1. **两载体共享同一套「母题语法」，只换皮肤**：左侧强调条、编号眉、箭头递进、✓/✗ 判定、胶囊徽章、深底示意图、"一句话收口"条——在 A 里是 CSS 组件，在 B 里是原生 shape 或烤进 PNG 的构图。母题是品牌的真正不变量，比色值更稳定。
2. **判定色语义完全同构**：绿（A: pine `#1E5A44` / B: 青绿 `#16C79A`）＝核心·正例·当前·已验证；暖色（A: clay `#B5532F` / B: 暖金 `#E0B84A` 兼任）＝警示·反例·情绪爆点；金（A: amber `#C8932E` / B: `#E0B84A`+`#FFC000`）＝金句·回环·价值。唯 **✗ 判定符的颜色按「面」分治（v1.1 定版）：浅面＝clay 暖红系，深面＝压灰 `#8A97A3`**（见 §1.5）。迁移新项目时先带走这套语义，再配色。
3. **B 的独有资产是"AI 插画配方"**（深夜蓝黑 + 青绿主光 + 暖金辅光 + 玻璃/金属/全息线框材质 + 等距或低角度全景），已可提炼为两段可直接投喂生图模型的提示词（见 §3.2/§3.3）。A 的独有资产是"深底 SVG 示意图规范"（dg-*），两者在"深底示意图"层面天然拼合（见 §4）。
4. **氛围铁律一句话**：深色页永远"一页一个光心、文字压暗部、辉光只给强调"。A 的 hero 用两枚径向光晕（右上绿 α.32 + 左下金 α.10），B 用"插画自带光源 + 垂直压暗纱（三停为主）"，配方不同、观感同宗。
5. **最小迁移集只有 10 件**（§6）：左强调条、mono/中文双轨编号、→ 箭头链、✓/✗ 双色判定、胶囊徽章、绿描边高亮框、终端窗、金句框、深底示意图五色节点、插画提示词。带走这 10 件，新项目即可"一眼知到"。

## 2. 图标风格规范（PPT 侧归纳）

> B 的图标几乎全部烤在 PNG 里，无矢量可考，以下为渲染图目测归纳（整体可信，线宽为估值）。A 侧仅 pill 内少量字符图标（↺/→/✦ 类），不构成图标系统。

1. **风格基调：线性（stroke-based）为主，面性为辅**。
   - 线性：约 1.5–2px 视觉线宽、圆头端点、几何简形（罗盘/星标/代码括号/灯泡/盾牌/文档/时钟/链接）。用于：章节议程卡条目、要点清单、功能列表。
   - 面性（filled）：只出现在**徽章类容器**内——圆底图标 chip（S17 输出栏蓝色系）、六边形白图标（S31）、圆底徽章条（S23 星/日历/对话）。规则：**容器越重，图标越实**。
2. **三种标准载体**（图标不裸奔，必有底座）：
   - **圆角方 chip**：青绿描边+深底，议程卡行首（S3/S9/S16 等章节卡）；
   - **圆底/圆环**：发光圆环底座（S29 步骤卡）、圆环+线性图标列表（S11 痛点/价值列）；
   - **六边形**：大概念专用（进/存/取、合规条款、三场分享编号）。
3. **颜色规则＝语义分区，不跟装饰走**：
   - 青绿 `#16C79A` 系＝主线概念/产品动作（默认色）；
   - 蓝 `#4A9EFF`/`#3B82F6`/`#4FC3F7` 系＝**外部引用、合规、中性流程**（S6 Karpathy 引用、S31 合规、S13 MVP 蓝变奏、S23 集成流程）——"蓝=这不是我的主观，是外部/客观"；此语义是否原设计有意无法从源文件判定，本册建议**将其转正为规则**；
   - 暖金 `#E0B84A` 系＝价值/机会（S11 金矿价值列）；
   - 暖红粉＝痛点/警示（S11 痛点圆环列）；冷蓝虚线＝痒点/可选（S11）；紫＝失控/反面点缀（S28 左半）。
4. **一页一族**：同页图标同风格同线宽同载体，不混线性与面性于同一清单。
5. **发光克制**：图标本体不发光，发光属于底座（圆环辉光、描边 glow），且只给当前强调项。

### 3.1 风格配方（可复述）

- **主题元素池**（按出现频次）：玻璃全息数据平台/电路城市（封面）、发光节点网络+悬浮文档卡（演示页）、等距发光旅程地图（目录）、等距工厂流水线/分拣机器人（概念隐喻）、蓝图线框建筑+工程师背影（架构/工地章）、矿洞金脉（价值隐喻）、机器人施工队沙盘（Agentic 章）、书本/沙漏/交接之手+光粒（情绪收尾章）、写实桌面+手机/键盘（互动指令页）。**规律：每章一个"世界观场景"，章内页共享该场景的材质与光。**

## 5. 氛围一致性 —— 深色页的光学规则

1. **光从哪来**：
   - A hero：双光源定式——**右上主光**（品牌绿 `rgba(30,90,68,.32)`，椭圆 680×420px @ 88%,-10%）+ **左下辅光**（品牌金 `rgba(200,147,46,.10)`，520×360px @ 6%,120%）。对角线布光，绿:金强度约 3:1。
   - B：光源在插画内（传送门光柱、金矿脉、引擎辉光、台灯），位置服从构图铁律"亮部避开文字区"（S38 违例：文字与亮部同侧居左，v1.1 定性禁止复制，见 §3.1）；页面层再加**垂直压暗纱**：90° 三停为主（38/41 页 ang=5400000，S7/S10/S20 反向 270°；S12/S13 两停、S21 四停），主流版式（约 S23–S41）顶 `#03070E` α100 → 中 `#06111A` pos 44–89% α0–59 → 底 `#03070E` α84；**α 按插画亮度逐页大幅调整**（顶 0–100、中 0–94、底 0–98），封面档见 §1.7。
2. **留黑比例**：暗部 ≥60%（插画提示词已写入）；**「文字覆盖区上方纱不透明度 ≥80%」自 v1.1 起升格为 QA 硬判据**——对每个文字框投影区取渲染图平均亮度，**L>60/255 即判失败**，重调渐变纱停靠点后重渲重测（PIL 可算；纱由 tokens_deck.py `add_veil()` 注入，重调即改 stops 参数；S38 即违例样本，禁止复制）。高密度页（表格/信息图）插画只在四角透光，中部近全黑。
3. **发光强度的度**：
   - 一页**一个光心**（S1 平台、S11 金矿脉、S38 引擎），其余光点做呼吸感点缀；
   - 辉光只给强调对象：当前项描边、核心节点、关键路径；正文容器一律哑光（`#142028` 实底/70% 玻璃，无 glow）；
   - 强调描边有粗细档表音量（v1.1 三档定版）：1.4pt（17780 EMU）金句级、**1.2pt（15240）品牌绿重点级**、1.0pt（12700）中性常规级——原片实测重点级 1.2–1.3pt 混用、S38 一处 1.0pt 绿边，历史测量保留为事实，新项目一律收口 1.2pt（S38 属源片例外，勿仿）。
4. **换色区隔**：特殊话题整页换副色（合规页转保险库蓝 S31、外部引用转蓝 S6、MVP 蓝变奏 S13），但**页面骨架三件套（竖条/标题/面包屑）永远保持品牌绿白灰**——换肉不换骨。
5. **A↔B 的氛围接口**：A 的深色嵌入块（hero/code/diagram/footer）就是 B 的"迷你页面"——在浅色文档里遇到需要氛围的内容，进深色块并沿用本节规则；反之 B 若需要"纸感"页（讲义/附录），可反向引入 A 的 `#F5F3EC` 纸底（目前 B 无此用法，属演绎建议）。

## 7. Do / Don't

### Do
1. **Do** 每个深色页只设一个光心，光源用品牌绿主、暖金辅（约 7:3），文字永远压在暗部。
2. **Do** 用描边颜色和粗细给容器"表态"：金 1.4pt 金句、绿 **1.2pt** 重点/当前（v1.1 定版；原片 1.0–1.3pt 漂移，S38 的 1.0pt 属源片例外，勿仿）、灰蓝 `#243441` 1.0pt 中性——容器底色统一深底（`#142028`，亮档 `#1B2735`）。
3. **Do** 对比页遵守完整语法：左灰右彩、✕/✓ 头、中缝 VS、底部"本质一句话"收口，四件缺一不可。
4. **Do** 把编号、路径、状态、图例一律放进"元信息层"（A 用 mono 大写宽字距，B 用面包屑/进度语/图内编号），与正文层字体或字级明确分离。
5. **Do** 生成插画时在提示词里写死色锚（#03070E 底/#16C79A 主光/#E0B84A 辅光）、暗部比例（≥60%）和留字方位，成图后仍叠垂直压暗纱兜底（参数按插画亮度逐页调）。
6. **Do** 特殊话题（合规、外部引用）整页换副色蓝以示"这页不一样"，但保留绿白灰页面骨架三件套。
7. **Do** 复用"分受众发糖"双卡收尾（绿=产品/第一受众，金=前端/第二受众），把每章的 takeaway 落到具体人身上。
8. **Do** 虚线只用于非承重内容：可选项、验收注、回环路径；实线/实底留给结论与核心。

### Don't
1. **Don't** 用 `#00B050`（Office 默认绿）做任何描边或强调——一切绿统一 `#16C79A`（A 浅底用 `#1E5A44`，深底亮档 `#6FBF95`）。B 现存恰两处（S2 徽章、S33 双受众卡描边），属待改正误用。
2. **Don't** 让 ✓/✗ 变成彩色 emoji，或让 ✓ 出现非绿色、✗ 出现非暖红色——判定双色是全品牌最硬的约定。
3. **Don't** 给正文容器、图标本体加辉光；发光只属于当前强调项与插画光心，处处发光等于没有重点。
4. **Don't** 在插画里生成大面积纯白、暖纸色或任何文字/水印；也不要把可维护的表格、清单文字烤进位图（B 的 37/39 页已被标为改稿风险，复杂图解才允许整图烘焙）。
5. **Don't** 在同一页混用两族图标（线性/面性、不同线宽、不同底座），或让图标颜色脱离语义分区自由配色。
6. **Don't** 破坏"留字暗区"：文字覆盖区上方压暗层透明度低于 80%、或亮部主体顶到文字下方，都不允许（B 现状仅 S38 一页违例，v1.1 起禁止复制；量化判据见 §5 条 2：文字框投影区平均亮度 L>60/255 判失败）。
7. **Don't** 把金色当普通强调随手用——每页金色落点 ≤1 处（关键词/金句/临门号召），多于一处金就贬值。
8. **Don't** 在浅色（纸面）区照搬深色母题的实色重底（如大块 `#142028`）；浅面强调走"三件套公式"（色字 + rgba 同色 .04–.1 底 + .2–.35 描边），深浅两面各守各的算法。
