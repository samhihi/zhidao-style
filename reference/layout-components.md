## 结论先行

1. **两载体是同一套「版式基因」的明暗两次表达**,同构核心可以压缩成四条:
   ① **左缘强调条 = 进入正题**(HTML `zd-call`/`zd-layer` 的 3–4px 左条 ↔ PPT 每页标题左侧 0.11×0.62in 青绿竖条);
   ② **描边即音量**(HTML 1px 中性线 → 3px 语义左条;PPT 1pt 隐形边 `#243441` → 1.2–1.4pt 绿/金强调边);
   ③ **元信息层与内容层严格分离**(HTML 用 JetBrains Mono 大写宽字距;PPT 用 12.5/11.5pt 面包屑小字 + 章节编号);
   ④ **对比必收口**(HTML `zd-concept` 后跟 `zd-call` 结论;PPT VS 页底部必有「灰引导 + 绿粗结论」条)。
2. **实现策略互补**:HTML 是「组件驱动」——约 30 个 `zd-*` 类拼装一切;PPT 是「版式驱动」——15 种页面原型(L1–L15) + 「原生文字骨架 + 整图烘焙图肉」双层分工。PPT 的设计 100% 在页内,母版/版式/主题三层全是出厂默认(41 页 rels 全指 slideLayout7 blank)——**迁移时这是最大技术债,新项目应把骨架收进母版层**。
3. **HTML 内部已经自带一个「深色模式胚胎」**:hero/code/diagram/footer 四类深墨嵌入块(`#181A1F` 底 + 亮档四彩)与 PPT 深底色板结构完全同构,是两载体之间现成的翻译层。
4. 迁移到新项目的**最小集 = 8 条 token 规格 + 5 个组件 + 5 种版式**(见 §6),其余都可以从这套最小集推导。

## 1. 空间系统

### 1.1 间距刻度(两载体各自梯度 + 对照)

**HTML(px)**——实测值可归并为一条「4 基准 + 若干半档」刻度:

| 档位 | 值 | 主要用途 |
|---|---|---|
| 微 | 9 / 10 | stack 层间 9;pill 群、kit 间隙 10 |
| 小 | 12 / 13 / 14 | 网格 gap(loop4 12、g3 14);call 内 flex gap 13 |
| 中 | 16 / 18 | 网格 gap(g2/stories/concept/sig 均 16);legend 16;mini/story/concept-card 内距 18 |
| 卡内距 | 15×18 / 16×18 / 18×20~22 / 22×24 | call `15px 18px`;layer `16px 18px`;sig `18px 20px`;changelog `18px 22px`;card `22px 24px`(tight `16px 18px`) |
| 大 | 24 / 28 / 34 | footer gap 24;版心水平 padding 28(≤560px→20);sec 纵距 `34px 0` |
| 版面级 | 44 / 54 / 60 / 72 / 90 | doc 上 44 下 90;shell 双栏 gap 54(全文档最大 gap);hero `72px 28px 60px` |

**PPT(EMU / in)**——刻度更粗,只有五档:

| 档位 | 值 | 主要用途 |
|---|---|---|
| 页眉锚点 | x=548640(0.60in)/ 777240(0.85in) | 竖条 x / 标题 x(竖条→标题留 0.14in) |
| 版心左缘 | x=868680(0.95in) | 正文容器、表格、结论条统一左缘;宽 10515600(11.5in)或 10469880(11.45in),左右对称 |
| 行节奏 | 640080–823595(0.70–0.90in)/ 685800(0.75in) | 表格行高 0.70/0.76/0.77/0.83/0.90in;目录清单行距 0.75in |
| 卡间缝 | 365760(0.40in) | 双受众卡中缝(S26 与 S33 几何完全一致,左卡右缘 5943600 → 右卡 x=6309360) |
| 区块间 | 内容区起点 y≈981k–1.8M(1.07–2.0in) | 随页型浮动,无严格刻度 |

**对照结论**:HTML 的刻度密(11 档),PPT 的刻度疏(5 档)——文档要承载密度层级,幻灯片只需要「页眉 / 版心 / 行 / 缝」。统一时建议以 **4px(HTML)/ 0.05in≈45720 EMU(PPT)为原子步长**,两边现值几乎全部落在该步长的整数倍上。

### 1.2 圆角刻度(两载体各自梯度 + 对照)

**HTML(px,14 档,规律「信息密度越小圆角越大」)**:

```
2(列表菱形弹头) 3(legend 色块) 5(表内 code) 6(chip/call ic/pchip)
7(zd-mark/nav a/kicker) 8(kit-item) 9(loop-path) 10(changelog/call)
11(SVG 普通节点 rx) 12(mini/tablewrap/code/layer/concept/story/step/sig)
13(SVG 核心节点 rx) 14(--r:zd-card/zd-road) 16(diagram/method) 999(胶囊)
```

归并为四级:**行内徽章 6–7 → 卡片 10–12 → 画布容器 14–16 → 胶囊 999**。

**PPT(roundRect `adj` 值,6 档;实际半径 = adj% × 短边,换算为推算值,待核渲染像素)**:

| adj | 实际半径(按实测短边推算) | 用例 |
|---|---|---|
| 3000 | ≈近直角 | S2 标注徽章 |
| 4000 | 0.132in(终端窗体,短边 3.3in)~ ≈0.17in(S19 画框,短边 4.2in) | S19 截图画框、S28 金句卡、S20/25/30 终端窗体 |
| 5000 | ≈0.06–0.15in 随短边(S38 结论条 ≈0.06 / S36 金句框 ≈0.10 / S41 QR 容器 ≈0.15) | S36/40 金句框、S38 结论条、S41 QR 容器 |
| 6000 | ≈0.10in(短边 1.7in) | S26/33 双受众卡 |
| 8000 | **仅 0.034in**(短边 384048=0.42in) | S20/25/30 终端标题栏 |
| 9000 | ≈0.06in(短边 0.65in) | 章节封面当前章高亮框 |

**对照结论**:PPT 的 adj 是「短边百分比」,同一 adj 在不同尺寸形状上半径不同,**档位数字与视觉半径甚至不单调**——adj=8000 的终端标题栏半径(0.034in)反而全场最小。这与 HTML 固定 px 圆角是两种制度。视觉落点上双方吻合在「卡片/窗体 ≈0.06–0.17in(约 6–16px)、行内小件近直角」。迁移建议:PPT 侧改用「目标像素反推 adj」而非固定 adj,锚定 HTML 的四级刻度。

## 2. HTML 组件库总表(全部 zd-*,按五类归档)

### 2.0 语义色变体的统一规律(先记公式)

- **强调三件套公式**:`实色文字 + rgba(同色,.04–.10) 底 + rgba(同色,.20–.35) 描边`——chip、concept、sig、loop-path、pill 全部由此派生。
- **三态语法**:同一组件的变体后缀即语义色——**pine=核心/肯定/已验证,amber=回环/心法/修订中,clay=警示/否定/原料**;第四色 moss(苔绿)只作 pine 的过程档,ink 灰作中性档。
- **对比度补偿铁律**:amber 在浅底作文字不得用原值 `#C8932E`(不达标),必须换深档——**v1.1 定版 `--amber-text: #7D5A0D`**(原片深档 #946A10/#9a6f12 对默认页底 paper 仍不达 AA,已降为考古注释);具体对比度数字以 color.md 为权威,本册只记结论。

## 7. Do / Don't

### Do

1. **左缘强调条开启一切正题**:HTML 组件级 3–4px、PPT 页面级 0.11in——这是品牌最强的版式签名,新载体(海报/长图/小程序)也从它长起。
2. **描边即音量,两档制**:中性容器用隐形边(1px line / 1pt #243441),只有「看这里」才升 1.2–1.4pt 品牌色边或 3px 左条;高亮永远稀缺。
3. **对比必收口**:任何 VS/正反内容,底部必须有一条「灰引导 + 品牌色粗体结论」——HTML 用 zd-call,PPT 用结论条,不许让听众自己总结。
4. **元信息层与内容层分族**:编号、路径、状态、图例、面包屑一律走 mono/小字大写宽字距(HTML)或 12.5/11.5pt 小字(PPT),绝不与正文混字面。
5. **旧态压灰、新态点亮**:进度(nav active / 章封面高亮)、前后对比(S19)、概念递进(S28 灰旧词→绿新词)全用同一语法——灰阶=过去,品牌绿=现在。
6. **保持「原生文字骨架 + 图肉」分层**:标题、面包屑、金句、结论、表格、出处这些「会改的字」必须留在原生层/活字层,图只承载不改的氛围与图解。
7. **每页/每区块只给一个金色焦点**:金(#E0B84A / amber)是「敲黑板」专用色,S5「信息/知识」、S14「不做什么」行都因稀缺而有效。
8. **宽内容自带滚动容器**(HTML tablewrap/svgscroll/pre 三件套),PPT 侧宽内容留在 0.95in 安全边内。

### Don't

1. **别把可维护文案烤进位图**:S37/39 的教训——改稿必须重生成图、不可检索、缩放发虚;新项目一切清单/条款/流程文字优先原生形状或 SVG。
2. **别引入色板外的强调色**:#00B050(S2 徽章、S33 卡边,slide XML 实测属实)、#4A9EFF/#2E8EFF/#3B82F6(蓝系仅存在于烘焙图内,slide XML 无从核,依 analysis 旁证)都是漂移;若要「换色区隔特殊话题」(S31 合规蓝),必须显式立项成 token,不能随手取色。
3. **别把胶囊圆角(999px)用在卡片上,也别把卡片圆角用在行内徽章上**:四级圆角各守其位——行内 6–7、卡 10–12、画布 14–16、胶囊 999;PPT 侧同一 adj 在不同尺寸上半径漂移(甚至不单调,见 §1.2),须按目标像素反推。
4. **别在浅底直接用 amber 原值(#C8932E)做文字**——必须换深档 `--amber-text`(v1.1 定版 #7D5A0D;原片 #946A10/#9a6f12 对默认页底不达 AA,降为考古,数字以 color.md 为权威);同理深底小字避免用低对比中间灰做关键信息。
5. **别给非当前项任何亮色**:章封面纪律是「非当前章整行 #8A97A3,连编号一起灰」;一旦给次要项上色,「你在这里」就失效。
6. **别再把 PPT 设计全放页内**:母版空白导致换模板一无所得、大纲视图/读屏拿不到结构;骨架(页眉带/结论条/占位符)应进 slideLayout。
7. **别破页眉字号恒定**:29pt 页题、12.5/11.5pt 面包屑、30pt 章名是全片节拍器,内容再挤也不缩标题(挤了就拆页,S13/14 一节两页是正确示范)。
8. **别让同一文案在复用页间漂移**:章封面第六章小注两版并存——「定义力,才稀缺」(S3/9/22/34)vs「想清楚,才稀缺」(S16/27),4:2 分布;复用模板时文案应来自单一数据源。

### 待核清单(本文所有不确定推断汇总)

1. PPT roundRect 实际渲染半径(adj% × 短边为推算,未逐一量渲染图);
2. 烘焙图内文字字号均为目测值(非矢量,XML 无据);

6. 现有 2048×1152 烘焙资产在 4K 放映下的表现。
